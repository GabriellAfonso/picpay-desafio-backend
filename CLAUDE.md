# CLAUDE.md — PicPay Clone

## Project

Simulates PicPay: registration (CPF/CNPJ), login, profile lookup, transfers.
Stack: Python 3.13 / Django 6.0.3 / DRF 3.16.1 / PostgreSQL 17 / pytest-django.
Run locally: `docker compose up --build` → port `8000`. `.env` at `dotenv_files/.env`.

---

## Architecture — Feature-Based

Organized by business domain. Each feature is a self-contained Django app (`features.<name>`).

```
features/<feature>/
├── models.py
├── urls.py
├── views/
├── services/       # one file per use case
├── repositories/   # single point of ORM access
├── serializers/
├── forms/
└── tests/
core/               # reusable code shared across features
├── exceptions.py
└── permissions.py
config/settings/    # base.py / dev.py / prod.py
```

**Rules:**
- Features must not import from each other. Use `core/` or signals.
- Views → Services → Repositories → Models. Never the other way.
- Views only call services. Services have no knowledge of `request`/`HttpResponse`.
- Repositories are the only layer that makes ORM queries.
- Domain errors: raise exceptions from `core/exceptions.py`, never `Http404` inside a service.
- Do not create a feature without a cohesive business domain.

---

## Code Rules

- All code in English (except strings visible to end users).
- Concise. No unnecessary abstractions. No duplicated logic.
- Type hints on public functions. PEP 8. Line limit: 100 chars.
- Models: always define `__str__`, `Meta.ordering`, `Meta.verbose_name`.
- `select_related`/`prefetch_related` — never queries inside loops.
- URLs with `app_name` and descriptive names.

---

## Security

- Never hardcode credentials. Use environment variables (`os.environ["KEY"]` — fails loudly if absent).
- Never disable `CSRF_COOKIE_SECURE`, `SESSION_COOKIE_SECURE`, `SECURE_SSL_REDIRECT` in production.
- All user input goes through a form/serializer with validation.
- `get_object_or_404` instead of `.get()` without handling in views.
- Explicit permissions on every authenticated view.
- `DEBUG = False` in production.
- Never `.raw()` or `format()` in SQL with user input.

---

## Tests

- All new code has tests. Do not skip.
- `pytest-django` with `@pytest.mark.django_db`.
- Test behavior, not internal implementation.
- `factory_boy` instead of JSON fixtures.
- Minimum: happy path + 1 error case per use case.

---

## Migrations

- Never edit a migration already applied in production.
- Generated by `makemigrations`, never written by hand (except data migrations).
- Data migrations: document the reason at the top of the file.

---

## Specific Notes

- Feature `picpay`: HTML views in `views/`, REST endpoints in `views/rest_views.py`. Keep separation.
- Login by **email**, not username. `EmailAuthenticationForm` in `core/forms.py`.
- **Guest Login** creates temporary users for demo — do not break when refactoring registration.
- CPF/CNPJ: `validate-docbr`, validators in `features/picpay/validators/`.
- Roles: `django-role-permissions` — assign in the registration service, never in the view.

## CI/CD Pipeline

```
commit → pre-commit (local) → CI (every push/PR) → CD (master, if CI green)
```

**Pre-commit:** `ruff --fix`, `ruff-format`, `bandit`.

**CI (`.github/workflows/ci.yml`):**
1. `ruff check`
2. `bandit`
3. `manage.py migrate --check`
4. `manage.py test`
5. `docker build`

CI env: `DJANGO_SETTINGS_MODULE=config.settings.test`, `SECRET_KEY=<any>`.

**CD (`.github/workflows/cd.yml`):** SSH → `git pull` → `docker compose up --build -d` → health check `GET /health/`.

GitHub Secrets: `VPS_HOST`, `VPS_USER`, `VPS_SSH_KEY`, `VPS_APP_PATH`.
